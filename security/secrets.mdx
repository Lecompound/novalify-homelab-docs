# 8. Identity & Secrets

## Current State

| Secret Type | Storage | Concern |
|------------|---------|---------|
| ASUS credentials | In-system memory | Single admin account, no rotation |
| Tailscale auth | Node keys | ✅ Key expiry supported |
| HA API token | Hermes config.yaml | ⚠ Plaintext in config |
| Obsidian API key | Hermes config.yaml | ⚠ Static key in config |
| Nabu Casa token | Hermes config.yaml | Cloud-based auth (acceptable) |
| Neural Memory | Local DB | No external auth needed |

## Gaps

| Gap | Severity | Detail |
|-----|----------|--------|
| No secrets rotation | MEDIUM | Keys/tokens have no rotation schedule |
| Secrets in config files | MEDIUM | API keys stored in plaintext YAML |
| No MFA everywhere | MEDIUM | HA and UniFi accounts lack MFA verification |
| No service accounts | LOW | No isolated accounts with least privilege |

## Recommendations

1. Implement **secrets rotation** (90-day cadence)
2. Consider **macOS Keychain** or **1Password CLI** for credential storage
3. Enable **MFA** on all management interfaces that support it
4. Use **Tailscale ACL tags** for service-to-service authentication
5. Document credentials in **Obsidian vault** (encrypted note)
