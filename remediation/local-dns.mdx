# 3. Local DNS Deployment

## Option 1: /etc/hosts (Quickest, Zero Dependencies)

No install required. Works on every OS immediately.

```bash
# Add to /etc/hosts (macOS/Linux) or C:\Windows\System32\drivers\etc\hosts
sudo tee -a /etc/hosts << 'EOF'
192.168.50.190  pve.ebg
192.168.50.169  ha.ebg
192.168.50.168  nvr.ebg
192.168.50.10   unifi.ebg ncp.ebg
192.168.50.145  controller.ebg
192.168.50.1    gw.ebg
192.168.50.149  hermes.ebg
EOF

# Verify
ping -c 1 pve.ebg
```

**Downside:** Must be done on every client. No DHCP propagation.

## Option 2: macOS dnsmasq (Full Local DNS)

### Check Port 53 Availability

```bash
sudo lsof -i :53 2>/dev/null || echo "Port 53 free"
```

macOS SIP may block binding to low ports. If blocked, use Option 1 or 3 instead.

### Install and Configure

```bash
brew install dnsmasq

# Add local domain records
cat >> /opt/homebrew/etc/dnsmasq.conf << 'EOF'
address=/pve.ebg/192.168.50.190
address=/ha.ebg/192.168.50.169
address=/nvr.ebg/192.168.50.168
address=/unifi.ebg/192.168.50.10
address=/ncp.ebg/192.168.50.145
address=/gw.ebg/192.168.50.1
address=/hermes.ebg/192.168.50.149
EOF

# Allow DNS through macOS firewall
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --add /opt/homebrew/opt/dnsmasq/sbin/dnsmasq

# Start service
sudo brew services start dnsmasq
```

### Point Clients at Local DNS

```bash
# macOS — per interface
networksetup -setdnsservers Wi-Fi 127.0.0.1 1.1.1.2
networksetup -setdnsservers Ethernet 127.0.0.1 1.1.1.2
```

### Push DNS to All Clients via ASUS DHCP

1. Login to ASUS admin at `https://192.168.50.1:8443`
2. Network → LAN → DHCP Server tab
3. Set **DNS Server 1**: `192.168.50.53` (Pi-hole IP when deployed)
4. Set **DNS Server 2**: `1.1.1.2` (Cloudflare fallback)
5. Click Apply → all DHCP clients receive the new DNS on lease renewal

## Option 3: Pi-hole LXC on PVE (Future State)

Post-Proxmox LAN restoration, deploy Pi-hole in an LXC for persistent local DNS, ad-blocking, and logging.

```bash
# On PVE
pveam update
pveam download local debian-12-standard_12.7-1_amd64.tar.zst
pct create 200 /var/lib/vz/template/cache/debian-12-standard_12.7-1_amd64.tar.zst \
  --storage local-lvm --memory 512 --swap 512 \
  --net0 name=eth0,bridge=vmbr0,ip=192.168.50.53/24,gw=192.168.50.1 \
  --hostname dns --unprivileged 1 --features keyctl=1,nesting=1
pct start 200
pct exec 200 -- bash -c 'curl -sSL https://install.pi-hole.net | bash'
```

## Verification

```bash
nslookup pve.ebg 127.0.0.1
nslookup ha.ebg 127.0.0.1
nslookup nvr.ebg 127.0.0.1
# All should return the correct IPs
```

## Rollback

- `/etc/hosts`: Remove added lines
- dnsmasq: `sudo brew services stop dnsmasq`
- Pi-hole: `pct stop 200 && pct destroy 200`
- ASUS DHCP: Set DNS back to `1.1.1.2` / `1.0.0.2`
- Client DNS: `networksetup -setdnsservers Wi-Fi empty`
