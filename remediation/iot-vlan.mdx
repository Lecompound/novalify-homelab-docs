# 5. IoT VLAN & Frigate Migration

## Current Problem

Frigate processes camera feeds from VLAN 40 (CAMERA) on the Home Assistant host which sits on VLAN 1 (MGMT). This means camera RTSP traffic crosses the router, lands on the management network, and exposes Frigate/HA services (`mqtt`, `samba`, `8123`) to whatever VLAN the camera traffic traverses. **This defeats the purpose of the camera VLAN.**

## Architecture Options

| Option | Isolation | Effort | Recommendation |
|--------|-----------|--------|---------------|
| **A: Frigate LXC on VLAN 1** | Weak | Low | Fallback |
| **B: IoT VLAN 30 + Frigate LXC** | **Strong** | **High** | **Best** |
| **C: Firewall rules only** | Moderate | Low | Quick fix |

## Prerequisites

- Proxmox LAN-reachable (see section 1)
- PVE host with `/dev/dri` (Intel QuickSync) or `/dev/apex` (Google Coral TPU)
- Debian 12 template downloaded in PVE storage

```bash
# Download template if not present
ssh root@100.89.90.38 "pveam update"
ssh root@100.89.90.38 "pveam download local debian-12-standard_12.7-1_amd64.tar.zst"
```

## Implementation — Option B (Recommended)

### Step 1: Provision VLAN 30 on ASUS

1. Login to ASUS admin at `https://192.168.50.1:8443`
2. Network → VLAN → Profile → Add
3. VLAN ID: `30`, Name: `IOT`
4. Subnet: `192.168.30.0/24`, DHCP ON, Range: `.100-.200`
5. DNS: `1.1.1.2` / `1.0.0.2`

### Step 2: Tag VLAN 30 on UniFi Trunk

Via UniFi Controller (https://192.168.50.10:8443):
1. Settings → Profiles → VLAN → New VLAN Profile: ID 30, Name IOT
2. Settings → Networks → New Network: Name IoT, VLAN 30, Subnet 192.168.30.0/24
3. The trunk port (Port 1) automatically carries VLAN 30 if set to "All"

### Step 3: Create Frigate LXC with Hardware Acceleration

```bash
# Create LXC on VLAN 30 with device passthrough
pct create 210 /var/lib/vz/template/cache/debian-12-standard_12.7-1_amd64.tar.zst \
  --storage local-lvm --memory 2048 --swap 1024 \
  --net0 name=eth0,bridge=vmbr0,ip=192.168.30.10/24,gw=192.168.30.1,tag=30 \
  --hostname frigate --unprivileged 1 \
  --features keyctl=1,nesting=1,mount=cifs;nfs \
  --cmode shell

# Hardware acceleration passthrough (on PVE host)
pct set 210 -dev0 /dev/dri
pct set 210 -dev1 /dev/apex   # only if Coral TPU present
pct set 210 -mp0 /dev/dri:/dev/dri

pct start 210
```

### Step 4: Install Frigate

```bash
pct exec 210 -- bash -c '
apt update && apt install -y docker.io docker-compose-plugin curl
mkdir -p /opt/frigate/config /opt/frigate/storage /opt/frigate/ssl
'

# Frigate requires Go2RTC for camera negotiation — Docker handles this
# Follow: https://docs.frigate.video/installation/advanced
```

### Step 5: Configure ASUS Firewall Rules

| Source | Destination | Port | Action | Purpose |
|--------|-------------|------|--------|---------|
| 192.168.30.0/24 | 192.168.40.0/24 | 554 (RTSP) | Allow | Camera streams |
| 192.168.30.0/24 | 192.168.40.0/24 | 80, 443 | Allow | NVR API/health |
| 192.168.50.0/24 | 192.168.30.10 | 22, 8971, 5000 | Allow | Mgmt access |
| 192.168.30.10 | 192.168.50.169 | 8883 | Allow | MQTT (TLS) |
| All other inter-VLAN | — | — | Deny | Default |

### Step 6: Update Frigate Configuration

```yaml
# /opt/frigate/config/config.yml
mqtt:
  host: 192.168.50.169
  port: 8883
  user: frigate
  password: "<generated>"
  tls_ca_certs: /ssl/ca.crt
  tls_insecure: false

cameras:
  front_door:
    ffmpeg:
      inputs:
        # Point at NVR RTSP endpoint
        - path: rtsp://192.168.40.168:554/Streaming/Channels/101
          roles:
            - detect
    ...

go2rtc:  # Required for stream negotiation
  streams:
    front_door: rtsp://192.168.40.168:554/Streaming/Channels/101
```

### Step 7: Update Home Assistant Integration

In Home Assistant:
1. Settings → Devices & Services → Frigate (remove existing integration)
2. Add integration → Frigate → URL: `http://192.168.30.10:5000`
3. Verify cameras and events appear

### Step 8: Validate Camera Streams

```bash
# From Frigate LXC
curl -s http://192.168.40.168/ | head -5  # Should work (VLAN 30 → 40)
ping -c 2 192.168.50.169  # Should work (VLAN 30 → 1, via firewall rule)

# From Management VLAN
curl -s http://192.168.30.10:5000/ | head -5  # Should work

# Stream validation
ffprobe rtsp://192.168.40.168:554/Streaming/Channels/101 2>&1 | grep -i "stream"
```

## Quick-Fix Alternative (Option C — Firewall Rules Only)

If the full LXC migration isn't feasible immediately:

```bash
# On HA host — restrict Samba to Tailscale IPs
echo "hosts allow = 100.64.0.0/10 127.0.0.1" >> /etc/samba/smb.conf
systemctl restart smbd

# On HA host — restrict MQTT with ACLs
cat > /etc/mosquitto/acl << 'EOF'
user frigate
topic readwrite cameras/#
user nova-bridge
topic read ha/#
topic write camera/#
EOF
echo "acl_file /etc/mosquitto/acl" >> /etc/mosquitto/conf.d/acl.conf
systemctl restart mosquitto
```

## Rollback

- Change VLAN back to 1 on UniFi port
- Restore Frigate to HA host (reinstall add-on)
- Remove firewall rules
- Revert ASUS VLAN configuration
